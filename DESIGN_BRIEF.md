# 합격각 — 디자인 브리프 (다른 창/디자인 작업용 핸드오프)

> 이 문서 하나만 보고도 디자인을 시작할 수 있게 정리한 것. **여기 적힌 데이터·색·화면 범위 안에서**
> 디자인하면 개발(안드로이드 Compose)에 그대로 얹힌다. 범위 밖(예: 없는 데이터)을 그리면 구현이 막힌다.

---

## 0. 이 앱이 뭔가 (한 줄)
취준생용 채용앱을 만들던 개발자가, **메가스터디 지원 포트폴리오**로 만드는
**정시 합격 분석 + 인스타식 학과 탐색 앱**. 가제 **"합격각"**.

- 내 수능(정시) 성적을 넣으면 → 지원 가능한 대학·학과와 **합격 가능성**(안정/적정/소신/상향)을 보여줌.
- 차별점이자 **핵심 축(hero)** = **인스타식 찾아보기**: 세로로 스와이프하며 "내 점수로 갈 수 있는 학과"를
  릴스처럼 한 장씩 넘겨봄.
- 참고 원본: 메가스터디 정시 합격예측 서비스 (성적입력 → 성적분석 → 대학검색·모의지원 → 합격예측 리포트).
- 플랫폼: **안드로이드 세로 모바일**(폭 360~412dp 기준). 시연용이라 깔끔·전문적 + 듀오링고풍 친근함.

## 1. 디자인 시스템 (이미 코드에 구현됨 — 이 토큰을 따라야 함)

**브랜드 톤**: 신뢰의 블루 × 따뜻함. 듀오링고 + 한국 모바일 UX. 폰트 **Pretendard**.
60% 웜화이트 배경 · 30% 블루 · 10% 포인트.

### 모양 토큰 (라운드 코너)
- 카드 18dp · 버튼 16dp · 작은버튼/로고 14dp · 칩 999(완전 둥근) · 바텀시트 위 24dp
- **버튼은 3D 깊이감**: 그림자 박스 + 본체 박스 2겹(듀오링고식). 눌리면 살짝 내려감.

### 마스코트
- 이전 채용앱 마스코트(시바견)는 **들어냄**. 입시앱 마스코트는 **새로 정하거나 생략 가능**.
  넣고 싶으면 합격/응원 톤의 캐릭터 1종(표정 몇 개) 제안해도 좋음. 필수 아님.

## 2. 화면 목록 (6 + 홈). 홈부터 시작 권장.

각 화면에 **표시 가능한 실제 데이터**를 같이 적었다. 이 필드들만 쓰면 바로 구현된다.

### 화면 A. 홈 / 대시보드 ★ 먼저
세로 스크롤. **6개 섹션 순서 확정**:
1. **헤더**: 앱명 "합격각" + **2027 수능 D-XXX** (D-day 카운트)
2. **내 성적 요약 카드**:
   - (성적 입력 전) "성적 입력하고 내 합격각 보기" 큰 CTA 버튼
   - (입력 후) 내 환산점수 / 전국 위치(상위 X%) / 유리 계열(인문 or 자연)
3. **★ HERO — "내 점수로 지원 가능한 학과"**: 인스타식 카드 1~3장 미리보기(가로 스크롤 or 큰 카드).
   각 카드에 **합격 라벨 배지**. 탭하면 → 풀 화면 찾아보기(화면 E)로. **시각적으로 가장 임팩트 있게.**
4. **빠른 메뉴 4개**(아이콘+라벨): 성적입력 · 대학검색 · 합격예측 · 찾아보기
5. **군별 추천 조합** 미리보기: 가군/나군/다군 각 1개 추천 학과 카드(합격 라벨 포함)
6. **성적대별 인기 학과 TOP5**: 순위 리스트(대학로고·학과·군·배치컷·모의지원수)

### 화면 B. 성적 입력 (정밀형, UI 편의 최우선)
입력 항목 (수능 영역):
- **국어**: 선택과목 칩(화법과작문 / 언어와매체) + 표준점수 · 백분위 · 등급
- **수학**: 선택과목 칩(확률과통계 / 미적분 / 기하) + 표준점수 · 백분위 · 등급
- **영어**: 등급(1~9)만 (절대평가)
- **한국사**: 등급(1~9)만 (절대평가)
- **탐구 1·2**: 과목 선택(사회탐구/과학탐구 중) + 표준점수 · 백분위 · 등급 (각각)
- UI 편의: 숫자 입력 쉽게(스텝퍼/큰 키패드), 선택과목은 칩 토글, 진행도 표시. "저장" 큰 버튼.

### 화면 C. 성적 분석
- 영역별 **백분위 막대/그래프**(국·수·영·탐)
- **유리 계열 판정**(인문/자연) + 한 줄 코멘트
- 전국 위치(상위 X%) / 총 환산점수 요약

### 화면 D. 대학 검색 & 모의지원
- 상단 **필터**: 군(가/나/다) · 계열(인문/자연) · 지역 · 배치컷 구간
- 검색창(대학명/학과명)
- 결과 **리스트 카드**: 대학로고 · 대학명 · 학과명 · 전형 · 군 배지 · 배치컷 · 경쟁률 · **합격 라벨 배지**
- 각 카드에 **"모의지원 담기"** 버튼

### 화면 E. ★ 인스타 찾아보기 (hero, 차별점)
- **세로 풀스크린 카드 1장 = 학과 1개**, 위아래 스와이프(릴스/틱톡식)
- 카드 내용: 대학로고·대학명 / 학과명 크게 / 군 배지 / **합격 라벨 + 합격확률%** / 배치컷 · 경쟁률 · 모집인원 / 지역 / 한 줄 소개
- 우측 또는 하단 **액션**: 관심(하트) · 모의지원 담기 · 상세보기
- 점수 기반 랭킹으로 추천(개발이 처리). 디자인은 "한 장씩 몰입" 느낌이 핵심.

### 화면 F. 합격예측 리포트
- 모의지원한 학과를 **군별(가/나/다)로 묶어** 표시 — 정시는 군당 1개만 지원하니 "내 조합" 개념
- 각 학과: 합격 라벨 + **합격확률%** + 배치컷 대비 내 점수 차이
- 군별 비교(어느 조합이 안정적인지)

## 3. 데이터로 쓸 수 있는 것 / 없는 것 (중요 — 없는 건 그리지 말 것)

**있음(모집단위마다)**: 대학명, 대학로고, 지역, 학과명, 전형명, 군(가/나/다), 계열(인문/자연),
모집인원, **배치컷(백분위)**, 경쟁률, 지원자 평균, (내 성적 입력 시) **합격 라벨 + 합격확률%**.

**없음(그리지 말기)**: 등록금/장학금, 취업률, 학과 상세 커리큘럼, 교수 정보, 합격 후기, 면접 정보.
→ 학과 "한 줄 소개"는 가능(자동 생성), 긴 상세 소개문은 placeholder.

**주의**: 지금은 **더미 데이터**(대학 10곳·학과 40개). 나중에 실데이터로 교체해도 화면은 그대로 동작.
숫자(배치컷 99.3 등)는 예시일 뿐, 디자인엔 "이런 값이 들어온다" 정도로만 반영.

## 4. 산출물 형식 (이렇게 주면 바로 구현)
- **화면 단위**로 Figma 프레임 또는 PNG 이미지. 화면당 1개(상태 분기는 따로: 예 "홈-성적입력전 / 홈-입력후").
- 위 색 토큰·합격 라벨 색·라운드/3D 버튼 규칙을 따를 것.
- **홈 → 찾아보기 → 성적입력** 순으로 주면 그 순서대로 개발.
- 폭 360~412dp(모바일 세로) 기준. 안전영역(상단 상태바·하단 제스처) 여백 고려.

## 5. 전체 작업 방향 (어긋나지 않게)
- 백엔드(Spring Boot + PostgreSQL)·데이터·합격판정 로직은 **개발 쪽에서 이미/계속 구축 중**.
  디자인은 **화면 UI만** 책임지면 됨.
- 핵심 셀링 포인트 2가지를 디자인이 살려야 함: ① **인스타 찾아보기(hero)** ② 메가 정시서비스의
  **정밀한 성적입력·합격예측** 흐름을 깔끔하게.
- 톤: 입시 스트레스를 줄여주는 **친근하고 명확한** 느낌(듀오링고풍) + 데이터 신뢰감(블루).
